## 📋 测试步骤

### 测试 1：创建 Pico LED 项目

1. 启动 CLI 工具
2. 选择 `1` (新建项目)
3. 选择 `1` (Raspberry Pi Pico)
4. 选择 `1` (LED 闪烁)
5. 引脚：`25` (或直接回车使用默认值)
6. 间隔：`2` (或直接回车使用默认值)
7. 项目名：`pico_led` (或直接回车使用默认值)
8. 确认：`y`

**预期结果：**
- 显示项目摘要
- 生成 Kiro 命令
- 命令复制到剪贴板
- 尝试打开 Kiro
- 提示在 Kiro 中粘贴命令

**生成的命令：**
```
使用 Arduino MCP server 的 full_workflow_led_blink 工具，
用 Raspberry Pi Pico 做一个 LED 闪烁，25 号引脚，每 2 秒闪一次
```

### 测试 2：创建 Uno LED 项目

1. 选择 `1` (新建项目)
2. 选择 `2` (Arduino Uno)
3. 选择 `1` (LED 闪烁)
4. 全部使用默认值（直接回车）
5. 确认：`y`

**生成的命令：**
```
使用 Arduino MCP server 的 full_workflow_led_blink 工具，
用 Arduino Uno 做一个 LED 闪烁，13 号引脚，每 1 秒闪一次
```

### 测试 3：查看帮助

1. 选择 `5` (帮助)
2. 查看帮助信息
3. 返回主菜单

### 测试 4：退出

1. 选择 `0` (退出)
2. 验证程序正常退出

## 🐛 已知问题

### 问题 1：Kiro 未自动打开
**原因**：Kiro 可执行文件未在标准位置  
**解决**：手动打开 Kiro，命令已复制到剪贴板

### 问题 2：剪贴板功能不可用
**原因**：pyperclip 未安装  
**解决**：手动复制显示的命令，或安装 pyperclip

## 💡 使用技巧

### 技巧 1：快速使用默认值
所有输入都有默认值，直接按回车即可使用

### 技巧 2：命令已复制
命令会自动复制到剪贴板，在 Kiro 中直接 Ctrl+V

### 技巧 3：手动复制
如果自动复制失败，命令会显示在屏幕上，手动复制即可
