我们成功实现了一个**自然语言驱动的 Arduino 开发 MCP Server**！

## 快速开始

### 1. 查看设计文档

```bash
plans/2026-02-01-arduino-mcp-server-design.md
```

完整的技术架构和设计思路。

### 2. 安装依赖

```bash
cd arduino-mcp-server
pip install -e .
```

### 3. 运行测试

```bash
python test_basic.py
```

验证代码生成和意图解析功能。

### 4. 安装 arduino-cli（可选）

如果要实际编译和上传：

```bash
# Windows
winget install ArduinoSA.CLI

# 配置
arduino-cli config init
arduino-cli core update-index
arduino-cli core install arduino:avr
```

### 5. 配置 Kiro MCP

在 Kiro 中添加配置（`.kiro/settings/mcp.json`）:

```json
{
  "mcpServers": {
    "arduino": {
      "command": "python",
      "args": ["-m", "arduino_mcp_server.server"],
      "disabled": false
    }
  }
}
```

参考: `arduino-mcp-server/examples/kiro-mcp-config.json`

### 6. 开始使用

在 Kiro 中直接对话：

```
"用 Arduino Uno 做一个 LED 闪烁，13 号引脚"
```

  pinMode(LED_PIN, OUTPUT);
  Serial.begin(9600);
  Serial.println("LED Blink Started");
}

void loop() {
  digitalWrite(LED_PIN, HIGH);
  Serial.println("LED ON");
  delay(1000);

## 核心洞察

### 为什么 AI 难搞定嵌入式？

1. **物理世界复杂性** - 硬件状态、时序、电气特性
2. **反馈链路长** - 编译→烧录→测试→调试
3. **上下文缺失** - 不知道硬件连接方式
4. **错误诊断难** - 软硬件问题难区分

### 我们的解决方案

> **让 AI 专注于理解用户意图，让工具链处理技术细节**

- AI 负责：理解需求、选择方案
- 工具链负责：代码生成、编译上传
- 两者结合：可靠 + 智能

## 性能指标
