## 测试用例

### TC1: 新建 Pico LED 项目
1. 启动 CLI
2. 选择"新建项目"
3. 选择"Raspberry Pi Pico"
4. 选择"LED 闪烁"
5. 输入引脚 25
6. 输入间隔 2 秒
7. 验证命令生成正确
8. 验证 Kiro 打开

**预期结果**: 
```
使用 Arduino MCP server 的 full_workflow_led_blink 工具，
用 Raspberry Pi Pico 做一个 LED 闪烁，25 号引脚，每 2 秒闪一次
```

### TC2: 新建 Uno LED 项目
1. 启动 CLI
2. 选择"新建项目"
3. 选择"Arduino Uno"
4. 选择"LED 闪烁"
5. 使用默认引脚 13
6. 使用默认间隔 1 秒
7. 验证命令生成正确

**预期结果**:
```
使用 Arduino MCP server 的 full_workflow_led_blink 工具，
用 Arduino Uno 做一个 LED 闪烁，13 号引脚，每 1 秒闪一次
```

### TC3: 输入验证
1. 输入无效的引脚号 (如 100)
2. 验证错误提示
3. 重新输入有效值
4. 验证接受

### TC4: 依赖检查
1. 启动 CLI
2. 验证 Python 检查
3. 验证 arduino-cli 检查
4. 验证 MCP Server 检查
5. 显示检查结果

## 风险和缓解

### 风险 1: Kiro 路径查找失败
**缓解**: 提供手动输入路径选项，显示命令让用户复制

### 风险 2: 剪贴板功能不可用
**缓解**: pyperclip 失败时仍显示命令，用户可手动复制

### 风险 3: 依赖未安装
**缓解**: 提供详细的安装指南和链接
